## Домашнее задание 2: Интеграция с API и работа с состоянием

### Цель
Подключить UI из первой домашки к реальным данным и довести конвертер до рабочего состояния: выбор валют, получение курса, расчет результата, обработка загрузки и ошибок.

### Стек
- `Vite + React`
- `MUI`
- `CSS Modules`
- `fetch` или `axios` для HTTP-запросов
- `Vitest + React Testing Library` для тестов

### API
- Использовать `https://api.frankfurter.dev/v2`
- Минимум нужны эндпоинты:
  - список валют;
  - получение курса/конвертации для выбранной пары.

### Что нужно реализовать
1. Получить с API список доступных валют и отрисовать их в оба селекта.
2. Реализовать ввод суммы в базовой валюте.
3. При изменении суммы или пары валют получать актуальный курс и показывать результат конвертации.
4. Добавить состояния:
   - `loading` во время запроса;
   - `error` при ошибке сети/сервера;
   - `empty` если данных нет.
5. Не допускать некорректные комбинации:
   - если в первом селекте выбрана валюта `X`, во втором нельзя выбрать ту же `X`;
   - при совпадении автоматически менять вторую валюту на первую доступную отличающуюся.
6. Реализовать кнопку `Swap` (поменять валюты местами).
7. Реализовать искусственную задачу с `key`:
   - при смене валютной пары сбрасывать локальное состояние дочернего компонента результата/описания через смену `key`;
   - в коде коротко обосновать, зачем здесь используется `key`.
8. Обновить блок `More about ...`:
   - заголовок формируется из текущей пары;
   - контент соответствует выбранным валютам (локальный словарь описаний или fallback-текст).

### Тесты
Минимальный набор тестов:
- сервисный тест для функции запроса (успех + ошибка);
- компонентный тест конвертера (рендер данных после успешного ответа API);
- тест на `Swap`;
- тест на reset по `key` (после смены пары состояние дочернего блока сброшено).

### Ограничения
- Не использовать state-менеджеры (`Redux`, `MobX`, `Zustand`) - достаточно `useState/useEffect`.
- Не использовать глобальные стили для фичи, только `CSS Modules`.
- Не хардкодить список валют, он должен приходить с API.

### Критерии приемки
- Проект запускается через `npm run dev`.
- Проект корректно билдится через `npm run build`.
- Тесты проходят через `npm run test`.
- Конвертация работает для разных валютных пар.
- Есть обработка `loading/error/empty`.
- Использование `key` для сброса состояния реализовано и видно в поведении UI.
