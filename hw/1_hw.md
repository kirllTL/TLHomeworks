## Домашнее задание 1: UI конвертера валют

### Цель
Сверстать страницу конвертера валют по макету и грамотно разбить интерфейс на React-компоненты. На этом этапе работаем только с UI и локальными данными.

### Стек
- `Vite`
- `React`
- `TypeScript`
- `Scss modules` для пользовательских стилей
- `Vitest + React Testing Library` для тестирования

### Что нужно реализовать
1. Поднять новый проект на `Vite`, `React`, `TypeScript`.
2. Сверстать экран конвертера валют `TODO  ДОБАВИТЬ ССЫЛКУ НА МАКЕТ`.
3. Добавить два селекта:
   - первый — валюта, из которой пользователь желает перевести;
   - второй — валюта, в которую пользователь хочет переводит.
4. Добавить два поля ввода:
   - первое — для указания суммы, которую нужно перевести;
   - второе — для отображения суммы после конвертации.
5. Добавить кнопку вида `More about {currency_1}/{currency_2}}`:
   - текст кнопки должен зависеть от выбранной пары валют;
   - по нажатию показывается/скрывается блок с кратким описанием выбранной пары.
   - блок с описанием изначально скрыт.
6. Все данные (список валют, тексты описаний, тестовые значения) хранить локально в коде.
7. Протестировать компонент с кнопкой `More about ...` и блоком описания через `Vitest + React Testing Library`.

### Ограничения
- Не использовать глобальные CSS-файлы для стилизации фичи (используем `*.module.css`).
- Основной фокус — структура компонентов, читаемость и переиспользуемость.

### Критерии приемки
- Проект корректно билдится без ошибок.
- Тесты запускаются и проходят через `npm run test`.
- Экран визуально соответствует макету из `README.md`.
- Компоненты разделены логично, без «монолитного» `App.tsx`.
- Кнопка `More about ...` работает и показывает описание выбранной пары.
- Используется `Scss Modules`.
- Для компонента `More about ...` есть тесты минимум на: скрыто по умолчанию, показ по клику, скрытие при повторном клике.
