## Домашнее задание 3: Persist State и пользовательские сценарии

### Цель
Сделать поведение конвертера устойчивым к перезагрузке страницы: сохранять важные пользовательские настройки в `localStorage`, корректно восстанавливать их и безопасно обрабатывать невалидные данные.

### Стек
- `Vite + React + TypeScript`
- `MUI`
- `CSS/SCSS Modules` (как в текущем проекте)
- `Vitest + React Testing Library`

### Что нужно реализовать
1. Сохранять в `localStorage` состояние конвертера:
   - базовая валюта (`from`);
   - целевая валюта (`to`);
   - введенная сумма;
   - состояние блока `More about ...` (открыт/закрыт).
2. При загрузке приложения восстанавливать это состояние из `localStorage`.
3. Добавить историю последних конвертаций (минимум 5, максимум 10 записей):
   - каждая запись: дата/время, пара валют, сумма, результат;
   - история хранится в `localStorage`;
   - новые записи добавляются в начало, старые вытесняются.
5. Сделать безопасную загрузку из `localStorage`:
   - если JSON поврежден или структура невалидна, приложение не падает;
   - используется fallback на значения по умолчанию.
6. Вынести логику персистентности в отдельный хук, например `usePersistentState`.

### Дополнительное задание (со звездочкой)
- Добавить версионирование данных в `localStorage` (`version` в объекте) и простую миграцию формата `v1 -> v2`.

### Тесты
Минимальный набор тестов:
- тест на восстановление состояния при старте;
- тест на сохранение состояния после изменения пары/суммы;
- тест на обработку невалидного JSON в `localStorage`;

### Ограничения
- Не использовать внешние библиотеки для persist (например, `redux-persist`).
- Не использовать глобальный стейт-менеджер только ради этой задачи.
- Не хранить в `localStorage` данные, которые всегда можно безопасно получить с API заново (например, полный список валют).

### Критерии приемки
- После перезагрузки страницы выбранная пара и сумма восстанавливаются.
- Блок `More about ...` восстанавливает последнее состояние (open/close).
- Приложение не падает при битых данных в `localStorage`.
- Тесты проходят через `npm run test`.
